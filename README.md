# 🗓️ Shifter App (Telegram Web App)

**Shifter** — це сучасний веб-застосунок, інтегрований безпосередньо в Telegram (Telegram Web App), створений для керування графіками роботи співробітників, відстеження KPI, створення задач та комунікації всередині команди роздрібних магазинів.

Додаток має сучасний, "iOS-подібний" дизайн з підтримкою світлої та темної тем, плавними анімаціями та інтуїтивно зрозумілою навігацією за допомогою плаваючої нижньої панелі (Bottom Tab Bar).

---

## ✨ Основний функціонал

Функціонал розділений на основі ролей (Глобальний Адмін, SM (Менеджер Магазину), RRP (Співробітник), User), забезпечуючи відповідний рівень доступу та конфіденційності даних для кожної ролі.

### 📅 Керування графіками (Редактор)
* **Інтерактивна сітка (Grid View):** Зручний перегляд змін на місяць з кольоровим кодуванням за часом (ранкові, денні, вечірні зміни).
* **Календар та Список:** Персональний вигляд графіку для кожного співробітника.
* **Кастомні зміни:** Додавання кастомного часу зміни через зручне нативне модальне вікно.
* **Статуси:** Відмітки про відпустки (🌴), лікарняні (💊), донорство (🩸) та видалення змін.
* **Закріплення годин:** Можливість закріпити підсумкову колонку з годинами при горизонтальному скролі.

### 📊 KPI та Дашборд
* **Дашборд:** Стартовий екран з компактною свайп-каруселлю, що відображає підсумок годин, прогрес KPI та статус сьогоднішньої зміни.
* **Візуалізація KPI:** Детальний перегляд показників ефективності.
* **Імпорт:** Можливість масового завантаження даних KPI з Excel/T-Sales для адміністраторів.

### 📌 Задачі та Комунікація
* **Задачі (Todo List):** Адміністратори та SM можуть призначати детальні завдання співробітникам (або групам).
* **Трекінг виконання:** Окремий розділ "Todo", де користувач бачить свої задачі. Підтримка підзадач (Subtasks), дедлайнів, маркування пріоритетів та зміни статусів виконання (В процесі, Виконано, Відкладено).
* **Новини (Announcements):** Публікація важливих повідомлень з підтримкою форматування (жирний, курсив, посилання) та прикріплення файлів.
* **Запити (Requests):** Співробітники можуть надсилати запити на вихідні або зміни в графіку, які менеджери можуть схвалювати або відхиляти.
* **Нотатки:** Створення особистих (бачить тільки власник і адмін) та публічних (бачать усі в магазині) нотаток. 

### 🏆 Гейміфікація та Досягнення
* **Автоматичні бейджі:** Система автоматично нагороджує працівників за досягнення:
  * 🛡️ **Сталевий імунітет** (за відсутність лікарняних)
  * 🔥 **Трудоголік місяця** (за найбільшу кількість відпрацьованих годин)
  * 🎖️ **Ветеран** (більше 1 року в компанії)
  * 🩸 **Донор** (якщо була зміна "Донорство")
  * 🦉 **Охоронець ТРЦ** (за нічні зміни/ремонти).
* **Кастомні бейджі:** Адміністратори (SM) можуть створювати власні бейджі з будь-якими емоджі та текстом, щоб відзначати особливі заслуги команди (наприклад, "Бетмен").

### 🎨 Персоналізація та Інтеграції
* **Колірні теми:** Вибір з декількох сучасних колірних палітр (Синя, Смарагдова, Пурпурова, Помаранчева, Рожева) з повною підтримкою Темного режиму.
* **Експорт в Календар (iCal):** Генерація унікального iCal-посилання для імпорту графіка змін у Google Calendar, Apple Calendar чи Outlook.
* **App Shortcuts (Швидкі дії):** Довге натискання на іконку додатка на головному екрані (Android/Windows) для миттєвого переходу до "Статистики" або "Задач".

### 🛠 Адміністративна панель
* **Глобальні налаштування:** Створення та видалення магазинів, налаштування матриці ставок.
* **Приховані форми:** Зручний інтерфейс створення сутностей (Новини, Задачі, Магазини) через модальні вікна, що викликаються спеціальними плаваючими кнопками (FAB).
* **Логування:** Відстеження дій користувачів у системі ("Хто, що і коли змінив").

### 🔐 Безпека та Авторизація
* **Хешування паролів:** Використання `bcrypt` для безпечного зберігання паролів.
* **Ізоляція даних магазинів:** Співробітники бачать графік, новини та нотатки тільки свого магазину.
* **Скидання паролю через Telegram:** Повністю автоматизований флоу відновлення паролю за допомогою Telegram-бота (\`/reset\` команда з генерацією одноразового токена).

### 🤖 Інтеграція з Telegram та Сповіщення (PWA)
* **Безшовна робота:** Додаток відкривається прямо всередині чату/групи за допомогою кнопки Menu Button або інлайн-клавіатури.
* **Стилізація:** Автоматичне підлаштування кольорів заголовка та фону до поточної теми користувача в Telegram.
* **Боти-сповіщення (Telegram):** Бот надсилає зведення та важливі повідомлення безпосередньо в призначені теми (Topics) групи. "Тихі години" враховуються.
* **Браузерні Push-сповіщення:** Повноцінна підтримка PWA Push-повідомлень (VAPID) з можливістю налаштування преференцій у профілі (Telegram, Push, або обидва).

## 🎨 Особливості UI/UX

* **Bottom Tab Bar:** Плаваюча навігаційна панель з іконками Дашборду, Графіку, Календаря, KPI та Профілю.
* **Glassmorphism:** Використання ефектів розмиття фону (backdrop-blur) для модалок, тостів та закріплених елементів.
* **Плавні переходи:** Top Progress Bar при первинному завантаженні, анімації `slide-up` для виклику панелей та модальних вікон.

## 🚀 Технологічний стек

* **Frontend:**
  * HTML5, CSS3, JavaScript (ES6 Modules)
  * [Tailwind CSS](https://tailwindcss.com/) (локальна компіляція для максимальної оптимізації)
  * Telegram Web Apps API (`telegram-web-app.js`)
* **Backend:**
  * [Node.js](https://nodejs.org/)
  * [Express.js](https://expressjs.com/)
  * База даних: [MongoDB](https://www.mongodb.com/) + [Mongoose](https://mongoosejs.com/)
* **Інше:**
  * `multer` (для завантаження файлів)
  * `bcrypt` (для безпеки паролів)
  * `axios` / Fetch API (REST комунікація)

## ⚙️ Встановлення та запуск (Локально)

1. **Клонуйте репозиторій:**
    \`\`\`bash
    git clone https://github.com/Stanistryit/shifter-app.git
    cd shifter-app
    \`\`\`

2. **Встановіть залежності:**
    \`\`\`bash
    npm install
    \`\`\`

3. **Налаштуйте змінні середовища:**
    Створіть файл `.env` у кореневій папці та додайте ваші конфігурації:
    \`\`\`env
    PORT=3000
    MONGO_URI=your_mongodb_connection_string
    TELEGRAM_TOKEN=your_telegram_bot_token
    TG_GROUP_ID=-1001234567890
    TG_TOPICS={"log_topic_id": "2", "news_topic_id": "4", "request_topic_id": "6"}
    NODE_ENV=development
    \`\`\`

4. **Запустіть сервер:**
    \`\`\`bash
    node backend/server.js
    # або через nodemon для розробки:
    # npm install -g nodemon
    # nodemon backend/server.js
    \`\`\`
